Pact

A Domain-Specific Language for Architecture Diagrams

Define software architecture with a simple DSL and generate beautiful diagrams automatically.

Read the Docs View Gallery

Supported Diagrams

Class Diagrams

Define components, types, relationships, and interfaces. Supports inheritance, composition, and dependency.

Sequence Diagrams

Visualize message flows between components with support for async calls, callbacks, and chains.

Flowcharts

Create flowcharts with conditionals, loops, and sequential steps. Perfect for process documentation.

State Diagrams

Model state machines with transitions, guards, and actions. Supports hierarchical and parallel states.

Quick Example

example.pact 
component OrderService {
    type Order {
        +id: string
        +items: OrderItem[]
        +status: OrderStatus
    }

    enum OrderStatus {
        Pending
        Processing
        Shipped
        Delivered
    }

    depends on Database
    depends on PaymentGateway

    provides OrderAPI {
        createOrder(items: OrderItem[]) -> Order
        getOrder(id: string) -> Order?
    }

    states OrderLifecycle {
        initial Pending
        final Delivered

        Pending -> Processing on submit
        Processing -> Shipped on ship
        Shipped -> Delivered on deliver
    }
}
